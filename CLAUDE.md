## Project Overview

**LirycVid+** is an AI music video studio that lets users create:
1. **Lyric videos** — upload a song, sync lyrics line-by-line, generate AI backgrounds, export a lyric video with animated on-screen text.
2. **Scene projects** — build a full music video shot-by-shot using an AI clip-per-scene workflow, then stitch clips into a final video with the uploaded song as master audio.

## Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 14 App Router |
| Backend / DB | Convex (real-time, file storage) |
| Styling | Tailwind CSS + CSS custom properties |
| UI primitives | Radix UI (hand-themed) |
| Animations | Framer Motion |
| AI generation | Replicate API (provider-agnostic wrapper) |
| Language | TypeScript throughout |

## Commands

```bash
# Install dependencies
npm install

# Start Next.js dev server (requires NEXT_PUBLIC_CONVEX_URL in .env.local)
npm run dev

# Start Convex backend (generates _generated/ files, hot reloads)
npx convex dev

# Type-check (Next.js app layer only — convex/ is compiled by Convex CLI)
npx tsc --noEmit

# Lint
npm run lint

# Deploy Convex schema to production
npx convex deploy
```

## First-time Setup

1. Copy `.env.local.example` → `.env.local`
2. Create a Convex project at https://dashboard.convex.dev → paste the URL and deploy key
3. Get a Replicate API token at https://replicate.com → paste it
4. Run `npx convex dev` — this generates `convex/_generated/` (real-time types, replaces stubs)
5. Run `npm run dev` in a second terminal

> `convex/_generated/` contains auto-generated stubs that are checked in. Running `npx convex dev` overwrites them with schema-aware types.

## Architecture Decisions

### Audio/Video Sync
The `useAudioSync` hook makes the `<audio>` element the single source of timing truth. A `requestAnimationFrame` loop reads `audio.currentTime`, updates `currentMs` state, and corrects video drift if it exceeds 150 ms. Play/pause/seek all flow through this hook. `useActiveLyricLine` derives the current lyric line from `currentMs` using a sorted binary search through `startMs` values.

### AI Generation
`convex/generation.ts` is the only file that calls Replicate. It's a Convex Action (Node.js runtime). The model IDs are constants at the top — swap them to change providers. The pattern is: create a `generationJobs` record → call Replicate → poll until complete → store output URL → update the project/scene record. All UI uses real-time Convex subscriptions so status updates propagate automatically.

### Data Flow
Convex real-time queries drive all UI state. Local React state is only used for unsaved form edits, which are auto-saved via `useDebouncedProjectSave` after 800 ms of inactivity.

### TypeScript Compilation
- `tsconfig.json` compiles `app/`, `components/`, `hooks/`, `lib/`, and `convex/_generated/` only.
- `convex/tsconfig.json` is used by the Convex CLI to compile backend files.
- The stubs in `convex/_generated/` satisfy TypeScript until `npx convex dev` generates the real typed versions.

## Design System

Design tokens are CSS custom properties defined in `app/globals.css`. Tailwind maps these via `tailwind.config.ts`. Key tokens:

```
--studio-bg       #080810   page background
--studio-surface  #0f0f1c   panel base
--studio-elevated #171728   raised cards
--studio-border   #1e1e30   borders
--accent          #7c3aed   primary violet
--ink-primary     #eeeef8   text
--ink-secondary   #8080a8   secondary text
```

Utility classes `surface`, `surface-elevated`, `panel`, `input-base` are defined in `globals.css` as `@layer utilities`.

## Key Conventions

- All Convex mutations/queries use the `api.*` object imported from `@/convex/_generated/api`.
- `Id<"tableName">` from `@/convex/_generated/dataModel` is used for all foreign-key props.
- Components never call Convex directly if a hook exists for that concern.
- The `"use client"` directive is required on any component using hooks, Convex, or browser APIs.
- `generation.ts` uses `"use node"` at the top (required for Convex Actions that call external APIs).
